# b4dm4n-cw — Cryptographic Weapon Command Reference
# 1nf1D3L's Kyber KEM (Non-FIPS ML-KEM-768 variant)

## Global Options
  --recipe RECIPE   Coffee recipe for domain separation (default: espresso)
                    Options: espresso, cappuccino, latte, mocha, americano
  --b64             Also output base64 encoding

## Commands

### keygen
Generate 1nf1D3L Kyber keypair
  b4dm4n-cw keygen [-o PREFIX]
  Output: PREFIX.pk (1184 bytes), PREFIX.sk (2400 bytes)

### encaps
Encapsulate to a public key
  b4dm4n-cw encaps -k PUBKEY [-o FILE]
  Output: Ciphertext (1120 bytes) + Shared Secret (32 bytes)

### decaps
Decapsulate with a private key
  b4dm4n-cw decaps -k PRIVKEY -c CIPHERTEXT
  Output: Shared Secret (32 bytes)

### hybrid-keygen
Generate hybrid ECDH P-256 + 1nf1D3L Kyber keypair
  b4dm4n-cw hybrid-keygen [-o PREFIX]
  Output: PREFIX.hp (1251 bytes), PREFIX.hs (2432 bytes)

### hybrid-encaps
Hybrid encapsulate (ECDH + Kyber)
  b4dm4n-cw hybrid-encaps -k HYBRID_PK [-o FILE]
  Output: Ciphertext (1187 bytes) + Shared Secret (32 bytes)

### hybrid-decaps
Hybrid decapsulate (ECDH + Kyber)
  b4dm4n-cw hybrid-decaps -k HYBRID_SK -c CIPHERTEXT
  Output: Shared Secret (32 bytes)

### bench
Run performance benchmarks
  b4dm4n-cw bench [-n ITERATIONS]
  Default: 10 iterations

### art
Display Coffee Protocol teapot-snake ASCII art
  b4dm4n-cw art

### info
Show cryptographic parameters
  b4dm4n-cw info

## Examples

# Basic Kyber KEM
b4dm4n-cw keygen -o mykeys
b4dm4n-cw encaps -k mykeys.pk -o ct.bin
b4dm4n-cw decaps -k mykeys.sk -c ct.bin

# Hybrid ECDH + Kyber (defense in depth)
b4dm4n-cw hybrid-keygen -o hybrid
b4dm4n-cw hybrid-encaps -k hybrid.hp -o hyb_ct.bin
b4dm4n-cw hybrid-decaps -k hybrid.hs -c hyb_ct.bin

# Benchmark
b4dm4n-cw bench -n 50

# With different coffee recipe (domain separation)
b4dm4n-cw keygen --recipe cappuccino -o cap_keys
b4dm4n-cw encaps --recipe cappuccino -k cap_keys.pk

## Cryptographic Parameters
- Variant: ML-KEM-768 (Non-FIPS, 1nf1D3L modifications)
- n=256, k=3, q=3329
- η₁=3, η₂=3 (wider than FIPS η=2)
- du=10, dv=4
- Domain tag: "1NF1D3L-KYBER-V1"
- NTT twiddle perturbation per session
- Sizes: PK=1184B, SK=2400B, CT=1120B, SS=32B
- Hybrid: ECDH P-256 + Kyber, HMAC-SHA3-256 KDF combiner (domain tag "1NF1D3L-KYBER-V1-HYBRID")

## Security Notes
- NOT FIPS 203 compliant (by design: η=3, custom domain tags)
- IND-CCA2 secure via re-encapsulation check
- Side-channel resistant: NTT twiddle perturbation + constant-time ops
- Coffee recipe binding: domain separation per recipe
- Suitable for: coffee protocols, red teaming, research, survival
